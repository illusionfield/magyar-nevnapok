# 0.6.0 kiadási jegyzetek

## Rövid összefoglaló

A `0.6.0` kiadás a projekt teljes szerkezeti modernizálását hozza el:

- egységes belépési pont,
- külön CLI wrapper,
- YAML-alapú artifactkészlet,
- deklarált pipeline,
- TUI,
- részletes auditriportok,
- és a teljes refaktorhoz igazított magyar dokumentáció.

## Mi változott a gyakorlatban?

### 1. Egyetlen elsődleges felület

A projekt dokumentált kezelőfelülete mostantól a `nevnapok` CLI. Ez azt jelenti, hogy a korábbi, egymást átfedő, nehezen követhető scriptfuttatások helyett egyetlen, következetes parancsrendszer maradt.

### 2. YAML lett az elsődleges strukturált formátum

A kézzel olvasott és szerkesztett artifactoknál a YAML lett az alapértelmezett forma. A rendszer továbbra is tud JSON-t olvasni, és igény esetén JSON testvérfájlokat is exportál.

### 3. A pipeline állapota látható és követhető

A `pipeline allapot` és a `output/pipeline/manifest.yaml` együtt világosan megmutatja:

- mi készült el,
- mi hiányzik,
- mi avult el,
- és melyik lépést kell legközelebb futtatni.

### 4. A refaktor lezárása nem állt meg a működő kódnál

Ebben a kiadásban nemcsak a kódszerkezet változott meg, hanem:

- a help szövegek,
- a dokumentáció,
- a riportelnevezések,
- a tesztek megfogalmazása,
- és a kommentelési szint is egységesítve lett.

### 5. Lint és erősebb karbantarthatóság

Bevezetésre került a helyi lintelés, hogy a refaktor után a szerkezeti tisztaság ne csak egyszeri állapot legyen, hanem folyamatosan fenntartható maradjon.

## Ajánlott első parancsok friss klón vagy cleanup után

```bash
  npm install
  npm run lint
  npm run ellenorzes
  npm run cli -- pipeline allapot
  npm run cli -- pipeline futtat teljes
```

## Kompatibilitási megjegyzés

A projekt továbbra is plain JavaScript + ESM irányban marad, és nem vezet be TypeScriptet. A hangsúly a könnyen olvasható, jól kommentelt, nyilvános minőségű kódbázison van.
