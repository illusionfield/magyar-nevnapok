# 0.6.2 kiadási jegyzetek

## Rövid összefoglaló

A `0.6.2` kiadás a friss TUI- és auditkör lezárását, valamint a repo-szintű
minőségellenőrzés formalizálását hozza el:

- elkészült a szabványos `npm run build`,
- bekerült a külön `npm run typecheck`,
- a TUI auditnézetei és az ICS-beállításnézet tovább finomodtak,
- a személyes primerforrás kezelése pedig végleg a saját szerkesztőnézethez került.

## Mi változott a gyakorlatban?

### 1. Van külön build parancs

A projekt teljes adatépítési folyamata mostantól szabványos npm scriptként is
elérhető:

```bash
  npm run build
```

Ez a teljes elsődleges pipeline-t futtatja, vagyis a régi
`nevnapok pipeline futtat teljes` rövid, megszokható megfelelője.

### 2. Van külön typecheck kör plain JavaScript projekthez

A projekt nem TypeScriptet használ, ezért a typecheck itt célzott, Node-alapú
statikus ellenőrzést jelent:

- minden saját `.mjs` / `.js` fájl szintaxisellenőrzése,
- a package által hirdetett entrypointok létezésének ellenőrzése,
- repo-szintű, gyorsan futó statikus sanity check.

A külön parancs:

```bash
  npm run typecheck
```

### 3. Az összetett ellenőrzés erősebb lett

Az `npm run ellenorzes` most már nemcsak lintet és teszteket futtat, hanem a
külön typecheck kört is.

### 4. TUI: részletesebb ICS és tisztább személyes primerworkflow

Az `ICS generálás – beállítások` nézet kapcsolónként részletesebb magyarázatot
ad, és a felső infósor azonnal leírja az aktuális érték gyakorlati következményét.

A kiadás idején a személyes primerforrás még külön TUI-s nézeten élt.
Azóta ezt a projekt egységes `Primer audit` workspace-be olvasztotta.

### 5. Böngészhető auditnézetek

A kiadás idején a végső primer és a primer nélkül maradó nevek auditjai külön,
böngészhető TUI nézetet kaptak. Azóta ezeket a projekt egyetlen, tabos
`Primer audit` munkatérbe vonta össze.

## Ajánlott ellenőrzések frissítés után

```bash
  npm install
  npm run lint
  npm run typecheck
  npm test
  npm run build
  npm run audit
```
