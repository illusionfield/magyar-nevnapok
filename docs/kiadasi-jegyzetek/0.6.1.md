# 0.6.1 kiadási jegyzetek

## Rövid összefoglaló

A `0.6.1` kiadás a friss refaktor utáni stabilizációs kör első lezárása:

- elkészült a primer nélkül maradó nevek külön auditja és TUI-s szerkesztési folyamata,
- visszakerültek a részletes ICS-opciók a dokumentált CLI-felületre,
- a scraper réteg pedig már Puppeteer 24 alatt is stabilan működik.

## Mi változott a gyakorlatban?

### 1. Új audit a primer nélkül maradó nevekhez

Az új audit havi bontásban mutatja azokat a normalizált és rangsorolt neveket,
amelyek végül sehol nem kerültek be a végső primerkészletbe. A riport külön közös
oszlopot is kapott, amely a két forrás unióját adja vissza.

### 2. Személyes primerkiegészítések külön helyi fájlban

A TUI-ból közvetlenül felvehető személyes primerkiegészítés a
`data/primary-registry-overrides.local.yaml` fájlba kerül. Ez nem követett, így
minden felhasználó saját naptárváltozatot tud építeni anélkül, hogy a közös
primerjegyzéket módosítaná.

### 3. Részletes ICS-vezérlés újra dokumentált, TUI-központú formában

A kiadás idején a `nevnapok kimenet general ics --help` még részletes, kapcsolóalapú
ICS-vezérlést használt, és a személyes primerforrás külön TUI-s nézethez tartozott.
Azóta a projekt ezt egységes, helyi YAML-profilra és közös `Primer audit` felületre
állította át; ez a bejegyzés a korabeli állapotot írja le.

### 4. Puppeteer 24 kompatibilitás és zöld audit

Az `npm audit` által jelzett sérülékeny Puppeteer-lánc megszűnt a `puppeteer 24.x`
frissítéssel. A HUN-REN HTTP-s forrást a projekt most központi kompatibilitási
launch-opciókkal védi ki a Chromium új HTTPS-first viselkedésével szemben, így a
scrape folyamat továbbra is működik fej nélküli módban is.

## Ajánlott ellenőrzések frissítés után

```bash
  npm install
  npm run audit
  npm run lint
  npm run ellenorzes
  npm run cli -- pipeline allapot
```
