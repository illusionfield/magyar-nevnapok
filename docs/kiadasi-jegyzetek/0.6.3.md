# 0.6.3 kiadási jegyzetek

## Rövid összefoglaló

A `0.6.3` kiadás a primer audit felület nagy újratervezését és a hozzá
kapcsolódó stabilizációs kört hozza:

- a Primer audit TUI most már audit-központú, többnézetes munkatér,
- a napi és név szerinti felülvizsgálat ugyanarra a közös view-modelre épül,
- a személyes primerbeállítások külön drawerbe kerültek,
- a riport napi végső primermezői helyesen töltődnek,
- a TUI pedig már a terminál magasságához is alkalmazkodik.

## Mi változott a gyakorlatban?

### 1. A Primer audit új információs architektúrát kapott

A régi, négyfüles megoldás helyett az új felület három fő módra épül:

- `Áttekintés`
- `Napok`
- `Nevek`

Ez azt jelenti, hogy ugyanazon auditadatból most külön nézet nyílik

- a gyors triage-ra,
- a napi döntések felülvizsgálatára,
- és a teljes névindex böngészésére.

### 2. Van teljes névindex az összes forrásból

A Primer audit most már nemcsak napokra, hanem nevekre is kereshető.
A névnézet exact névalakokkal dolgozik, és megmutatja, hogy egy név

- mely forrásokban szerepel,
- mely napokon fordul elő,
- volt-e végső primer,
- hiányzó jelölt,
- rejtett név,
- vagy helyi kijelölés.

### 3. Egységesebb lett a navigáció

A korábbi kontextusfüggő billentyűviselkedés helyett az új primer auditban
következetesebb a billentyűmodell:

- `Tab` vagy `1`–`3` — módváltás,
- `/` — keresés,
- `f` — szűrők,
- `s` — rendezés,
- `b` — személyes primer-beállítási drawer,
- `Enter` — drill-down vagy ugrás,
- `Space` — helyi kijelölés kapcsolása, ha ténylegesen akciózható név van fókuszban.

### 4. A napi végső primermezők hibája megszűnt

A primer audit riport `finalPrimaryNames` és `finalPrimaryCount` mezői most már
hiánytalan napoknál sem ürülnek ki hibásan.

Ez azért fontos, mert az új TUI és a CLI részletes nézete is ezekre a napi
összegző mezőkre támaszkodik.

### 5. A felület most már a terminálmérethez igazodik

A Primer audit TUI mostantól figyeli a terminál magasságát:

- a hosszú listák ablakolt megjelenítést kapnak,
- a kijelölt elem környezete marad látható,
- kisebb viewporton a részletek tömörebb megjelenítésre váltanak.

Ez megszünteti azt a hibát, amikor a nézet egyszerűen kifutott a terminál
aljáról.

## Ajánlott ellenőrzések frissítés után

```bash
  npm install
  npm run lint
  npm run typecheck
  npm test
  npm run build
  npm run audit
```
